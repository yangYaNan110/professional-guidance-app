# doc-summarize Skill - 专业介绍生成

## 功能描述

根据专业名称和类别，生成结构化的专业介绍，采用时间线叙述格式。

## 触发词

介绍、总结、概述、讲解、解读

## 核心能力

1. 专业类别检测（工学、理学、医学、法学等）
2. 时间线格式内容生成
3. 热门专业自动识别
4. 缓存策略优化

---

## 时间线格式规范

所有专业介绍必须采用**时间线叙述格式**，包含以下5个核心部分：

```
**起源与发展**  
[专业名称]的源头可追溯至[具体时间]的[具体事件/人物]。[关键里程碑]标志着[专业]进入[新阶段]。

**挫折与争议**  
[专业]发展历程中经历过[具体挫折/争议]。[影响]。[应对措施]。

**重大突破**  
[年份]，[具体事件]是[专业]史上最伟大的[突破/成就]。[后续影响]。

**现状与爆发**  
当前，[专业]正处于[发展态势]。[具体数据/案例]。[代表性成就]。

**未来展望**  
[专业]未来将在[方向]取得[预期成就]。[人才需求]。[职业前景]。
```

## 热门专业检测

```python
HOT_MAJORS = {
    # AI相关
    "人工智能", "机器学习", "深度学习", "自然语言处理", "计算机视觉",
    # 新兴技术
    "数据科学与大数据技术 "物联网", "", "云计算",区块链", "网络安全",
    # 热门工程
    "计算机科学与技术", "软件工程", "电子信息工程", "自动化",
    # 热门经济管理
    "金融学", "经济学", "工商管理", "会计学",
    # 热门医学
    "临床医学", "口腔医学",
    # 热门法学
    "法学"
}

def is_hot_major(major_name: str) -> bool:
    for hot in HOT_MAJORS:
        if hot in major_name or major_name in hot:
            return True
    return False
```

## 缓存策略

| 专业类型 | 缓存时间 | 说明 |
|---------|---------|------|
| 热门专业 | 24小时 | AI、金融、计算机等 |
| 普通专业 | 72小时 | 其他专业 |
| 实时数据 | 无缓存 | 分数线、录取数据 |

---

## 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| major_name | string | 是 | 专业名称 |
| category | string | 否 | 专业类别（自动检测） |
| related_majors | array | 否 | 相关专业列表 |

## 使用示例

```python
from doc_summarize import MajorSummarizer

summarizer = MajorSummarizer()

# 生成专业介绍
result = summarizer.generate(
    major_name="人工智能",
    category="工学"
)

print(result.success)
print(result.introduction)
print(result.is_hot_major)
print(result.cache_ttl_hours)
```
